# QA Checklist — Portal Initial Setup v1.0.0

## For the Project Designer (Human Testing)

### Build and Start
- [ ] Run `docker compose up -d` from the repo root — the Portal should build and start without errors
- [ ] Check the logs (`docker compose logs portal`) — you should see "Starting PlayRooms Portal server..." followed by "Portal server listening on port 3001"
- [ ] If `SHARED_SECRET` is not set (or empty), the Portal should exit with a FATAL error about the missing secret

### Health Check
- [ ] Open `http://localhost:3001/api/health` in a browser — you should see a JSON response with `"status": "ok"` and `"version": "1.0.0"`
- [ ] The `connectedInstances` should be `0` (no Host connected yet)

### Host Connection (requires PlayRooms Host)
- [ ] Configure the Host addon with `portal_url` pointing to the Portal and `portal_secret` matching the Portal's `SHARED_SECRET`
- [ ] Start the Host — the Portal logs should show "HA instance registered: ..."
- [ ] The health endpoint should now show `connectedInstances: 1`

### Guest Connection (requires Host + Share Link)
- [ ] Create a share link on the Host, then open the Portal URL with the share link token
- [ ] The guest should be able to connect through the Portal to the Host
- [ ] Check the health endpoint — `connectedGuests` should increment

### Error Cases
- [ ] Try connecting with a wrong secret — the connection should be rejected
- [ ] Try accessing `/api/join/invalidtoken` — should return a 503 or 404 error

## For the QA Tester (Technical Testing — Claude in Chrome)

### Build Verification
- [ ] Verify `portal/server/package.json` has only the expected dependencies: `express`, `socket.io`, `cors`, `nanoid` — no `buttplug`, `better-sqlite3`, `drizzle-orm`, or `socket.io-client`
- [ ] Verify `portal/server/tsconfig.json` compiles successfully: `cd portal/server && npx tsc --noEmit`
- [ ] Verify the Dockerfile does NOT download Intiface Engine, install `bluez`, `dbus`, `libudev1`, or `libusb-1.0-0`

### Relay Types Sync
- [ ] Compare `portal/server/src/shared/relay-types.ts` with the Host repo's `playrooms/server/src/shared/relay-types.ts` — they should be identical except for the header comment
- [ ] Verify `RELAY_PROTOCOL_VERSION = 1` matches in both repos

### API Endpoints
- [ ] `GET /api/health` returns `{ status: "ok", mode: "portal", version: "1.0.0", connectedInstances: 0, connectedGuests: 0 }`
- [ ] `GET /api/portal/info` returns `{ enabled: true, mode: "portal", version: "1.0.0", connectedInstances: 0 }`
- [ ] `GET /api/join/badtoken` returns 503 when no Host is connected

### Configuration
- [ ] Verify config.ts reads from `/data/options.json` (HA mode) and falls back to env vars
- [ ] Verify the three config options: `port` (default 3001), `shared_secret` (required), `log_level` (default "info")
- [ ] Verify `PORTAL_SECRET` and `RELAY_SECRET` env vars are accepted as fallbacks for `SHARED_SECRET`

### Socket.IO Namespaces
- [ ] Verify `/relay` namespace exists and requires `instanceId` + `secret` in handshake auth
- [ ] Verify default namespace accepts guest connections with `roomId`, `token`, `name` query params
- [ ] Verify `isHost=true` connections are rejected on the default namespace

### Rate Limiting
- [ ] Verify `MAX_EVENTS_PER_SECOND = 50` in guest-namespace.ts
- [ ] Verify rate limit counter resets every 1000ms

### Grace Period
- [ ] Verify `config.gracePeriodMs` defaults to 60000 (60 seconds)
- [ ] Verify guests receive "Host is temporarily offline" on Host disconnect
- [ ] Verify guests are disconnected with "Host has gone offline" after grace period expires

### HA Addon Structure
- [ ] `repository.yaml` exists at root with correct name, url, maintainer
- [ ] `portal/config.yaml` has slug `playrooms-portal`, version `1.0.0`, port 3001
- [ ] `portal/build.yaml` targets `ghcr.io/home-assistant/amd64-base-debian:bookworm`
- [ ] `portal/run.sh` reads HA options via `bashio::config`
- [ ] `portal/translations/en.yaml` has descriptions for all three options

### Version Consistency
- [ ] `portal/config.yaml` version matches `portal/server/package.json` version (both `1.0.0`)
- [ ] `/api/health` version matches `package.json` version

### Log Output
- [ ] Logs use format: `ISO_TIMESTAMP [LEVEL] [Subsystem] message`
- [ ] Subsystems are limited to: Portal, Relay, Guest, API
- [ ] `setLevel()` is called with `config.logLevel` in index.ts

### Code Quality
- [ ] No imports from Host-specific modules (lobby, chat, toybox, buttplug, drizzle, etc.)
- [ ] All relay type imports reference `../shared/relay-types.js`, not `../types/index.js`
- [ ] No `PORTAL_MODE` flag — the server runs as Portal unconditionally
